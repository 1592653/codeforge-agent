# CodeForge Agent вҖ” Architecture

## Overview

CodeForge Agent is a multi-agent system for automated code review and refactoring. It uses 5 specialized AI agents orchestrated through a sequential pipeline with shared context management.

## Agent Communication

Agents communicate through two mechanisms:

1. **Shared Context**: All agents read from and write to a central context store. Downstream agents access upstream outputs via `context.get_agent_output("agent_name")`.

2. **Message Passing**: Agents can send structured messages to each other for fine-grained coordination.

## Token Management

Each agent tracks its own token usage through a shared `TokenCounter`. The counter provides:
- Per-run tracking
- Daily budget management with alerts
- Cost estimation
- History export

## Configuration

The system is configured through:
- `config/settings.yaml` вҖ” Pipeline and agent settings
- `.env` вҖ” API keys and environment variables
- CLI flags вҖ” Per-run overrides

## Key Design Decisions

1. **Sequential pipeline** rather than parallel agents вҖ” ensures data dependencies are met and simplifies error handling.

2. **AST + LLM hybrid analysis** вҖ” static metrics provide reliable baselines while LLM adds semantic understanding.

3. **Auto-rollback on test failure** вҖ” the Validator agent can revert changes if tests fail, providing a safety net.

4. **Pluggable LLM backends** вҖ” supports Anthropic, OpenAI, and local models via Ollama.
